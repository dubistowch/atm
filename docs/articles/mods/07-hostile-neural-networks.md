# Hostile Neural Networks 新手教學與玩法筆記

## 簡介
Hostile Neural Networks 是一款以「模擬擊殺怪物來量產掉落物」為核心的模組。它的理念類似經典模組 Deep Mob Learning：你會收集資料、訓練某種「資料模型」，然後在機器中進行模擬戰鬥，產出對應怪物的掉落物。整合包中它常扮演「替代刷怪塔」或「補足稀有掉落供應」的角色，特別適合伺服器不鼓勵大規模刷怪塔、或世界怪物生成受限的情境。

## 模組特色
- 以「資料模型」與「模擬」取代長時間的實體刷怪。
- 玩法通常是：取得模型 → 餵資料/訓練 → 進行模擬 → 產出掉落。
- 支援用資料包新增自訂資料模型（官方 README 提到可用 datapack 與 JSON）。
- 產出型玩法對伺服器友善：較不依賴大量實體怪物堆積。
- 能把危險或麻煩的掉落（例如特定怪物素材）轉成可控產線。
- 適合和儲存/物流模組搭配，做成完整的自動化材料供應鏈。

## 按鍵操作
Hostile Neural Networks 主要是方塊/機器與物品介面操作，通常不依賴固定鍵位。

- 請到「選項 → 控制設定」搜尋：`Hostile`、`Networks`、`Neural` 等關鍵字，確認整合包是否新增快捷鍵。
- 若未找到，代表主要互動以右鍵開啟方塊介面、在 GUI 中設定為主。

## 如何設定
常見調整方式（依版本可能不同）：

- **模組設定檔**：通常位於 `.minecraft/config/`。
- **資料包擴充模型**：官方說明可用 datapack + JSON 新增自訂資料模型。
  - 世界資料包路徑：`你的存檔/datapacks/`
  - 伺服器請確保資料包與伺服器端一致。

如果你的整合包有加入額外資料模型（支援其他模組怪物），也多半是透過資料包擴充而來。

## 如何獲得模組道具
由於不同版本與整合包會調整配方/流程，以下以常見取得路線整理；請用 JEI/REI 查你版本的實際名稱與配方。

- **合成取得**
  - 資料模型（用於對應某種怪物/生物的模擬）
  - 進行模擬的機器方塊（模擬室/運算/製造類方塊，名稱依版本可能不同）

- **戰鬥與資料收集**
  - 通常需要擊殺對應怪物來收集訓練資料，提升模型等級/準確度。

- **產線化**
  - 當模型成熟後，就能用機器穩定產出該怪物掉落物。

代表性物品/方塊（3–6 個，名稱依版本可能不同）：
- 資料模型
- 用於訓練/儲存資料的裝置或道具
- 模擬用機器方塊
- 掉落物輸出/製造機器方塊

## 好玩的點在哪
- **把稀有掉落做成「白名單供應」**：先鎖定整合包常卡的素材（例如某些怪物器官、骨粉、火藥等），優先做出對應模型。
- **用小規模起步**：先做一條只供應一兩種關鍵掉落的產線，再慢慢擴展模型庫。
- **搭配儲存與物流**：把模擬產出直接接進倉庫系統，設定過濾、分流與自動補貨。
- **伺服器友善**：不需要堆一堆怪物與實體掉落，通常更穩定。
