# Hostile Neural Networks 新手教學與玩法筆記

## 簡介
Hostile Neural Networks（HNN）是一種「不用蓋大刷怪塔，也能量產怪物掉落」的玩法模組：你先用實際擊殺收集資料，訓練出某種怪物的「資料模型」，再把模型放進機器進行模擬，穩定產出該怪物的掉落物。

在整合包中它常見用途：
- 取代或補足刷怪塔（伺服器不鼓勵堆怪時特別好用）
- 量產特定素材（例如某些稀有怪物掉落）
- 把「危險/麻煩/不想刷」的素材變成可控產線

## 模組特色
- 以「資料模型 → 訓練 → 模擬」的流程取代大量實體刷怪。
- 對伺服器較友善：通常比堆大量實體怪物更穩定、更少路徑計算。
- 常可透過資料包擴充支援的怪物模型（多見於整合包作者/伺服器端調整）。
- 很適合接到倉儲/物流模組，變成「白名單材料供應」。

## 按鍵操作
HNN 主要以方塊/機器 GUI 操作為主，通常不依賴固定鍵位。

- 打開「選項 → 控制設定」，搜尋：
  - `Hostile`
  - `Neural`
  - `Networks`

若沒有找到鍵位也正常：表示主要互動是右鍵開 GUI、把物品放進格子、設定輸入輸出。

## 遊戲內設定與調整
1) **先用 JEI/REI 釐清你這包的流程**
- 不同整合包常改：模型製作成本、訓練需要的擊殺數、模擬機器的耗能與產量。
- 建議先查三件事：
  - 「模型怎麼做」
  - 「訓練靠什麼累積」
  - 「模擬機器需要什麼能源/耗材」

2) **機器 GUI：把它當成可控產線在調**
- 常見你需要在 GUI 內確認/調整的項目：
  - 目前模型等級/進度（是否已達可量產）
  - 輸入需求（能量、耗材、資料類物品）
  - 輸出掉落（是否需要接箱子/管線）

3) **Mod 設定畫面（若有）**
- 依整合包與版本可能不同：
  - `ESC` → **Mods** → Hostile Neural Networks → `Config/設定`（若有）
  - 若沒有 GUI，一般玩家也不需要改。

> 補充（只留短註）：若你看到「可用資料包新增模型」的說法，那通常是整合包作者/伺服器端在做的擴充；玩家端以遊戲內可用內容為主。

## 如何獲得模組道具
以「從 0 到量產」的生存路線整理（名稱依版本略有差異，配方以 JEI/REI 為準）：

- **做出第一個資料模型（Data Model）**
  - 通常會對應某一類怪物（例如殭屍/骷髏/烈焰使者…）。

- **先手動打幾隻怪收集訓練資料**
  - 建議挑「你本來就會刷」的怪當第一個模型，最不痛。

- **做出模擬機器並接上輸出**
  - 先從單一模型量產開始，確認耗能、產量、倉儲是否跟得上。

代表性內容：
- Data Model（資料模型）
- 訓練/資料累積用物品（依版本）
- Simulation/Prediction 類機器方塊（依版本命名）

## 好玩的點在哪
- **把稀有掉落做成「白名單供應」**：先鎖定整合包常卡的素材（例如特定怪物的器官/粉末/鱗片之類），優先做出對應模型。
- **小規模起步最舒服**：先讓它穩定供應 1～2 種核心材料，再擴到更多模型。
- **接進倉庫後像開外掛**：掉落自動進系統、缺料時再回頭擴產，不用每天追著怪跑。
