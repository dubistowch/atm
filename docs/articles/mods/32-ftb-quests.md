# FTB Quests（任務書）玩家教學：開啟、追蹤、交付、領獎與卡關排查

FTB Quests 在整合包裡不是「可有可無的成就系統」——它更像**導航 + 教學 + 進度記錄器**。

你會用它來：
- 知道「下一步該做什麼」（尤其是新手開荒）
- 把大目標拆成一小步一小步（中後期材料海）
- 在多人服把進度同步到同隊（避免 A 做完、B 還在原地重做）

> 這篇用**玩家視角**寫：不要求你背按鍵、不要求你懂任務設計。照著做，你就能把任務書當成「主線導覽 + 待辦清單」。

---

## 30 秒先記住 3 件事（超重要）
1) **先確認你會「打開任務書」**（不要猜按鍵，去 Controls 搜）
2) 任務完成有兩種：
   - **Detect / 自動偵測**：東西在你身上/你做過就會亮
   - **Submit / 手動交付**：你要在任務介面按「提交」把物品交出去
3) **完成 ≠ 領獎**：很多任務要你再按一次「Claim / 領取獎勵」

---

## 1) 怎麼打開任務書（不要猜按鍵：用 Controls 搜關鍵字）

不同整合包可能改過鍵位，請直接用遊戲內確認：

1. 打開：**選項（Options）→ 控制設定（Controls）**
2. 在搜尋框輸入以下任一個關鍵字（挑你找得到的）：
   - `Quest` / `Quests`
   - `FTB` / `FTB Quests`
   - `Open Quest Book` / `Quest Book`

你要找的動作通常長得像：
- **Open Quests** / **Open Quest Book**（打開任務書）

> 找到後把它記下來，或改成你順手的鍵位（並注意衝突提示）。

---

## 2) 介面速讀：你看到的每一塊是什麼（文字圖解）

任務書介面大致可以拆成這幾區（不同包 UI 會略有差異，但概念一樣）：

```
┌───────────────────────────────────────────────┐
│ [章節列表/分頁]  [搜尋]            [隊伍/設定] │
├───────────────┬───────────────────────────────┤
│ 章節/主線樹狀   │  任務內容區                   │
│ - Welcome      │  ┌───────────────┐           │
│ - Getting ...  │  │ 任務標題/描述  │           │
│ - Mainline ... │  │ 任務條件(Tasks)│           │
│               │  │ 獎勵(Rewards)  │           │
│               │  └───────────────┘           │
└───────────────┴───────────────────────────────┘
```

建議閱讀順序：
1) 先看任務標題與描述（通常會提示 Detect 還是 Submit）
2) 再看 Tasks（完成條件）
3) 最後看 Rewards（獎勵是否可選、是否每人可領）

---

## 3) 追蹤 / 釘選任務：把「下一步」黏在螢幕上

新手最常發生的事是：任務看完→關掉→忘記要做什麼。

解法：把任務釘起來。

打開任務後，找以下類型的按鈕：
- `Pin` / `Pinned`（釘選）
- `Track` / `Tracking`（追蹤）
- 📌 / 👁 圖示（依材質包可能不同）

> 實用技巧：一次只追 1～3 個任務。追太多，你會把 HUD 變垃圾訊息牆。

---

## 4) 任務怎麼算完成：Detect vs Submit（最常卡的核心）

### A) Detect（自動偵測）
你把目標物品放在背包（或依任務設定，在特定容器/狀態），任務就會自己打勾。

**通常不會扣物品。**

### B) Submit（手動交付）
任務要求你「把物品交出去」：
1) 物品先在你身上
2) 打開任務
3) 在 Tasks 區找到 `Submit / 交付 / 提交`
4) 按下去，物品才會被扣掉並計入進度

---

## 5) 交付物品（Submit）完整流程（照著做就不會卡）

- [ ] 把物品先放在**背包**（不要先放箱子）
- [ ] 打開任務書 → 點進該任務
- [ ] 在 Tasks 區找到物品條目
- [ ] 確認它有 `Submit` 按鈕
- [ ] 按下 `Submit`
- [ ] 最後到 Rewards 區按 `Claim / 領取`

文字圖解：

```
[Tasks]
- [物品圖示] 需要：X 個
  狀態：0 / X
  按鈕：Submit / 交付
```

---

## 6) 領獎（Claim）：完成不代表你已經拿到東西

完成任務後，去看 Rewards 區，常見會有：
- `Claim`（領取）
- `Claim All`（全部領取）
- 可選獎勵（讓你在多個獎勵中選一個）

避免選錯的原則：
- 不確定就先不要領（很多包允許你晚點領）
- 或先用 JEI 查獎勵物品用途

---

## 7) 多人服最常見卡關排查（不同步 / 偵測不到 / 重置）

### 情境 A：我跟隊友明明一起做，為什麼進度不同步？
1) **確認你們在同一個 FTB Team**（不要只靠口頭）
2) **重登一次**（快取/同步延遲很常見）

### 情境 B：任務寫 Detect，但我拿著物品/做完了，任務沒反應
按順序排查：
- [ ] 先確認它是不是 Submit（Tasks 有沒有 Submit 按鈕）
- [ ] 把物品放到背包主格再拿起來一次
- [ ] 丟地上再撿起來（重新觸發偵測）
- [ ] 離開任務頁再點回來
- [ ] 重登
- [ ] 多人：確認你在正確隊伍

### 情境 C：我想重置任務／任務被重置了怎麼辦？
- 先確認你沒切隊/離隊
- 若需要真正重置，多半要管理員用版本對應的指令/工具處理
- 請先在聊天輸入：`/ftbquests help` 以你遊戲內顯示的指令為準（不要照網路文章硬背）

---

## 8) 最快自救表（卡住就照這張做）

### 我不知道下一步做什麼
- 回到主線章節
- 釘選 1 個最接近你現況的任務
- 缺材料就用 JEI 查用途/配方

### 我做完了但沒完成
- 先判斷 Detect vs Submit
- Submit 就去按提交
- Detect 就用「放背包主格 / 丟地上撿起 / 重登」刷新偵測

### 多人不同步
- `/ftbteams` 確認同隊
- 重登
